# Research Conference Slide QC Checklist

Last updated: 2026-07-08

Use this before finalizing a research-conference or professor-facing deck.

## Narrative Order

- [ ] Actual completed work and decisions appear before exploratory device/collaboration ideas.
- [ ] Grant A/B decisions and clinical needs are visible early enough.
- [ ] Device collaborations are framed as exploratory, secondary, feasibility, or education-application candidates unless a verified protocol says otherwise.
- [ ] The deck answers: what happened, what was achieved, what remains, and what happens next.

## Formal Names

- [ ] Event names are searched and written formally when available.
- [ ] Short nicknames are not the only visible event label when an official name and theme exist.
- [ ] If only a short name is available, the slide marks the formal name as `要確認` in notes or QC memo.

Example pattern:

`JACS-NEXT関東甲信越支部勉強会「肺区域切除入門〜エキスパートから学ぶビデオクリニック〜」`

Use an abbreviated display title only if the formal name is preserved in a subtitle, note, or reference.

## Numeric Reporting

Each progress-report slide should include at least one concrete number when available:

- participants
- sessions
- applications
- grants
- meetings
- interviews
- documents
- deadlines
- next artifact date

Avoid abstract wording such as `教育資産化した` unless it is tied to a concrete event, participant count, material count, or next use case.

## Grant Slides

- [ ] Reference date is visible or in the notes.
- [ ] Deadline-past grants are not in `これから出す候補`.
- [ ] A still-actionable grant with supervisor-email context is reported separately.
- [ ] Gmail draft is labeled `下書き作成済み/送信未実施`, not sent.
- [ ] Official deadline and email discussion are separated.

## Visual And Layout Checks

- [ ] One slide has one message.
- [ ] One item is emphasized: a number, status, decision, deadline, or next artifact.
- [ ] Real-use photos are used when they improve understanding.
- [ ] Product-only photos are secondary when real-use photos exist.
- [ ] Visible source-system labels are removed from audience-facing cards unless the slide is an evidence table.
- [ ] Tiny labels and awkward short line breaks are fixed.
- [ ] Footer page numbers and reference slides are checked after reordering.

## Professor-Facing Q&A

When anticipating strict questions, answer with:

1. the protected main line,
2. what will be removed or downgraded,
3. concrete dates,
4. the next artifact to be delivered.

Avoid answers that only state good intentions.
