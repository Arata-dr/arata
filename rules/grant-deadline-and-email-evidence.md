# Grant Deadline And Email Evidence Rules

Last updated: 2026-07-08

## Core Rule

Every grant table or handout must state the reference date.

Example:

`2026-07-08時点`

At that reference date, already-closed grants must not appear in the main `これから出す候補` or `推奨` section.

## Required Columns

Use these columns unless the user asks for a different format:

| Column | Meaning |
|---|---|
| `助成金/公募名` | Official name. Use the official page, not a nickname. |
| `参照日` | Date when deadline and status were checked. |
| `締切/募集期間` | Official deadline or application period. |
| `公式URL` | Official page or official PDF. |
| `適格性` | Age, position, field, institution, or theme restrictions. |
| `必要書類` | Form, recommendation, CV, budget, signature, upload method. |
| `現在ステータス` | Candidate, draft, submitted, receipt confirmed, etc. |
| `メール証跡` | Supervisor/recommender discussion, draft, sent, recommendation request, or no evidence. |
| `次アクション` | The next concrete action and date. |

## Deadline Handling

- If the deadline is in the future, the grant may appear in `これから出す候補`.
- If the deadline has passed, move it to a separate section:
  - `締切後`
  - `過去対応`
  - `提出対応済み`
  - `受付証跡要確認`
- Do not keep deadline-past grants in the main recommendation table for future work.
- If the application site has not opened yet, write `募集開始後に要確認`.

## Email Evidence

Before reporting grants to a supervisor or in a research conference, search email for the supervisor/recommender name plus each grant name or recognizable abbreviation.

Use separate labels:

| Email evidence | Label |
|---|---|
| Discussion exists | `メールやりとりあり` |
| Draft exists only | `下書き作成済み/送信未実施` |
| Sent email exists | `送信済み` |
| Recommendation request exists | `推薦依頼済み` |
| No current-run search | `Gmail未確認` |

Email evidence does not prove the official deadline, eligibility, or submission receipt. It only proves discussion, draft, or sent status.

## Common Failure To Avoid

Do not make a broad grant handout that mixes:

- already-closed grants
- still-actionable grants
- grants discussed with a supervisor
- grants with no email context
- submitted items
- draft-only items

Instead, split them by status and evidence strength.

## Recommended Output Sections

1. `今から動く候補`
2. `募集開始待ち`
3. `提出対応済み/受付証跡要確認`
4. `締切後のため現在候補から除外`
5. `メール証跡のある案件`
6. `次の48時間でやること`
